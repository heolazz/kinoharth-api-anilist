Authentication & Access

To interact with the AniList proxy API, your client requests should include correct authorization headers if configured. Cross-Origin Resource Sharing (CORS) is enabled for allowed origin domains.

Header Name Required Expected Value Description
X-KinoHarth-Key Conditional <your_api_key> Verifies the client is authorized to request the AniList gateway proxy.
Content-Type Yes application/json Defines request payload structure.

API Routing Reference

Trending Anime

GET

Fetch the list of currently trending anime series sorted by popularity and activity.

/api/trending
Query Parameters
page Page number (default: 1)
perPage Items per page (default: 20, max: 50)

Popular Anime

GET

Fetch the most popular anime of all time sorted by total rating score and member counts.

/api/popular
Query Parameters
page Page number (default: 1)
perPage Items per page (default: 20)

Search Anime

GET

Perform direct text query search on the AniList database for titles, tags, and synopses.

/api/search?q=Frieren
Query Parameters
q / search Search keyword (Required)
page Page number (default: 1)

Anime Details

GET

Fetch complete details for a specific anime series, including trailers, characters, and related media.

/api/anime/:id
Path Parameters
id AniList Anime ID (e.g. 173518 for Frieren)

Anime Catalog & Filters

GET

Advanced catalog explorer supporting detailed filters for genres, release format, and media status.

/api/catalog?genre=Action&status=RELEASING
Query Parameters
genre Genre name (e.g. Action, Fantasy)
status MediaStatus (e.g. RELEASING, FINISHED)

Airing Schedule

GET

Fetch the list of airing episodes within a specified Unix timestamp start and end range.

/api/schedule?start=1700000000&end=1700086400
Query Parameters
start Unix timestamp start (Required)
end Unix timestamp end (Required)
